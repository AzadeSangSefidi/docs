مستندات API سرویس ما

معرفی API

API سرویس ما یک رابط برنامه‌نویسی RESTful است که به شما امکان می‌دهد با سرویس ما تعامل داشته باشید. این مستندات تمام نقاط پایانی موجود، پارامترهای مورد نیاز و پاسخ‌های ممکن را پوشش می‌دهد.

# پایه URL برای تمام درخواست‌ها
BASE_URL = "https://api.example.com/v2"

ویژگی‌های کلیدی

  • پشتیبانی از JSON برای تمام درخواست‌ها و پاسخ‌ها
  • احراز هویت از طریق توکن JWT
  • مکانیزم استاندارد برای خطاها
  • پاسخ‌های سریع و بهینه
  • مستندات جامع با نمونه‌کدهای آماده

احراز هویت

برای دسترسی به اکثر نقاط پایانی API، شما نیاز به یک توکن احراز هویت دارید. این توکن باید در هدر تمام درخواست‌ها ارسال شود.

POST /auth/login

دریافت توکن دسترسی

برای دریافت توکن دسترسی، نام کاربری و رمز عبور خود را به این نقطه پایانی ارسال کنید.

پارامترهای درخواست

نام نوع الزامی توضیحات
username string بله نام کاربری شما
password string بله رمز عبور شما

مثال درخواست

POST /auth/login HTTP/1.1
Host: api.example.com
Content-Type: application/json

{
    "username": "your_username",
    "password": "your_password"
}

مثال پاسخ

HTTP/1.1 200 OK
Content-Type: application/json

{
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires_in": 3600,
    "token_type": "Bearer"
}

استفاده از توکن

پس از دریافت توکن، آن را در هدر Authorization تمام درخواست‌ها قرار دهید:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
